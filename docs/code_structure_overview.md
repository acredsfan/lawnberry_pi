## Architecture / Protocols

| Path | Purpose | Subsystem | Callable interfaces |
| `backend/src/core/client_identity.py` | Validates the immediate peer and carries one canonical client IP across the local frontend-to-Uvicorn hop. Generic forwarding headers remain untrusted; the internal header is accepted only from loopback, so direct LAN callers cannot rotate or collapse production quota identities. | Security/runtime | `client_ip(connection) -> str \| None`; constant `CANONICAL_CLIENT_IP_HEADER`. |
|---|---|---|---|
| `backend/src/protocols/mission.py` | Structural Protocol (runtime-checkable) that defines the minimal interface `NavigationService` requires from `MissionService`, breaking the circular import between the two services without any implementation changes. | Architecture / Protocols | Protocol class `MissionStatusReader` (`@runtime_checkable`): attribute `mission_statuses: Mapping[str, Any]` (read-only `mission_id → MissionStatus` map); `async update_waypoint_progress(self, mission_id: str, waypoint_index: int) -> None`. |

## Frontend views and router

These are Vue components and routing; they provide UI and don’t expose reusable function APIs. Notable UI-only behavior includes `frontend/src/views/LoginView.vue` accepting the explicit deployment credential by default, offering configured username/password as a separate mode, and never advertising a known default; `frontend/src/views/ControlView.vue` restoring manual-control authorization from an existing authenticated session, rendering password+TOTP unlock flows correctly, and surfacing camera stream fallback plus richer lockout/controller-health detail while tolerating both wrapped and flat camera API payloads. `frontend/src/views/MissionPlannerView.vue` now surfaces recovered-paused mission detail, waypoint progress, mission create/start operator hints, a persisted mission-planner-specific map-style/source selector, source-specific alignment calibration, and canonical mower body-center telemetry. The E2E harness in `frontend/src/main.ts` now exposes the mission store alongside auth/map/control stores for regression scenarios.

`frontend/src/components/map/BoundaryEditor.vue` also exposes the parcel-helper workflow in the maps UI: imported helper layer, editable confirmed boundary, generated safe boundary with a distinct additional inset, source-specific display transform support, custom imagery source rendering, and a polled/status-driven blade-off drive-to-confirm checklist whose Confirm action is enabled only after backend arrival.

- Views: `frontend/src/views/*.vue` (DashboardView, ControlView, MapsView, MissionPlannerView, PlanningView, RtkDiagnosticsView, AIView, SettingsView, DocsHubView, LoginView)
- App shell: `frontend/src/App.vue`
- Router: `frontend/src/router/index.ts`
- Entrypoint: `frontend/src/main.ts`
